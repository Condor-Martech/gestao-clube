## Conventions

- Routes: app/(auth)/ (public), app/(dashboard)/ (protected), app/api/ (route handlers)
- Server Actions: `_actions.ts` per feature directory
- Components: `_components/` colocated under each route
- DB types: `types/database.types.ts` (generated via `npm run db:types`)
- Entity types: `types/entities.ts` (until db:types is wired)
- snake_case for Supabase tables/columns
- pt-BR strings live in `messages/pt-BR.json`
- shadcn components in `components/ui/`
- Forms: react-hook-form + @hookform/resolvers/zod

## Compact Rules (auto-injected by orchestrator)

These rules are injected as `## Project Standards (auto-resolved)` into every
sub-agent prompt that touches the matching code or tasks. The orchestrator
reads this section once per session and caches it.

### next-best-practices (app router, RSC, server actions)
- Default to Server Components. Add `'use client'` ONLY when needed (state, effects, browser APIs).
- Server Actions: validate inputs with Zod, return typed result, never trust client.
- Fetch data in RSC; never use `useEffect` to load data on the client.
- `params`/`searchParams` are async (`await params`).
- Use `next/image` for all images, `next/font` for fonts.
- `typedRoutes: true` — pass `Route` objects, not raw strings, to `<Link>`.

### react-best-practices
- Co-locate state with where it's used. Lift only when needed.
- Avoid prop drilling > 2 levels — use composition or context.
- Memoize ONLY when there's a measurable problem, not preemptively.
- Server-first data: prefer RSC over client fetching with TanStack Query.

### react-hook-form
- Use `useForm` with `zodResolver(schema)` from `@hookform/resolvers/zod`.
- Use `<Form>` shadcn primitives, not raw `<form>`.
- For server actions: pass form values to action, handle pending with `useFormStatus`.

### zod
- Define schemas in `lib/validators/` or colocated next to the action.
- Use `z.infer<typeof schema>` to derive TS types.
- `.safeParse()` over `.parse()` when handling user input.

### tailwind-v4-shadcn
- Tailwind v4 uses `@theme` directive in CSS, NOT `tailwind.config.js`.
- Dark mode via `next-themes` with `<ThemeProvider attribute="class">`.
- `cn()` from `lib/utils.ts` for conditional class merging.

### supabase-postgres-best-practices
- ALWAYS define RLS policies for new tables — never leave a table unprotected.
- Use `timestamptz` not `timestamp` for any time column.
- Use generated columns / triggers for derived data, not application logic.
- Index foreign keys and any column used in WHERE / ORDER BY frequently.
- Use views for read patterns that join multiple tables.

### vitest
- `vitest run` for CI / one-shot, `vitest` for watch mode.
- Use `@testing-library/react` for components, `@testing-library/user-event` for interactions.
- Mock with `vi.mock()`, NOT manual stubs.
- For RSC: extract logic into pure functions and unit-test those, do NOT render server components.

### playwright-best-practices
- Use Page Object Model for non-trivial flows.
- Mock external APIs at the network layer with `page.route()`.
- Run with `--ui` for debugging, headed mode for visual checks.
- Tag tests: `@smoke`, `@critical`, etc., for filtering.
